Script autónomo (create-admin.js) para criação do primeiro utilizador administrador inserido.

Middleware de autorização (checkRole) para restringir o acesso a rotas com base no cargo (ex: 'admin') implementado.

Rotas do painel de administração (/admin/users) definidas e protegidas para acesso exclusivo de administradores.

Sistema de CRUD (Criar, Ler, Atualizar, Eliminar) para gestão de utilizadores implementado (Controller, Service e Model).

Páginas de visualização (Views) para listar todos os utilizadores e formulário para criar/editar utilizadores adicionadas.

Helpers do Handlebars (eq, neq, lookup) para lógica condicional nas views adicionados.

Link para "Gerir Utilizadores" adicionado ao dashboard, visível apenas para administradores.

Correção de segurança aplicada para impedir que um administrador edite ou elimine o seu próprio perfil no painel de gestão.

Correção de dados aplicada para exibir corretamente o status "Ativo" (Sim/Não) na lista de utilizadores, corrigindo valores NULL no banco de dados.
